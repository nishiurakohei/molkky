

# 📋 Ankkaタイトル戦 アプリケーション仕様書

本アプリケーションは、モルックのタイトル戦（順位決定戦および本戦）を管理するためのWebアプリです。Reactでフロントエンドを構築し、Firebase Firestoreをバックエンドとしてリアルタイムなデータ同期を実現しています。

## 1. システム構成・技術スタック
* **Frontend**: React (18), Babel (Standalone)
* **Backend**: Firebase Firestore (リアルタイムDB)
* **CSS**: CSS Variablesを使用したレスポンシブデザイン（ダークモード対応）

---

## 2. コード構造と機能マップ

コードは大きく分けて「設定」「ロジック」「コンポーネント」の3層で構成されています。

### ① 設定・定数エリア (`Logics` セクション)
| 定数名 | 内容 | 変更時の影響 |
| :--- | :--- | :--- |
| `INITIAL_RATE` | 新規選手の初期レート(1500) | 選手の初期立ち位置が変わります。 |
| `ADMIN_PASSWORD` | 管理画面パスワード("kyoto") | 管理ログイン時のパスワードが変わります。 |
| `DEADLINE` | 試合入力期限 | この日時を過ぎると、試合結果の送信ができなくなります。 |

### ② 計算ロジック関数
* **`calcWinGain / calcLoseLoss`**: イロレーティングに基づいたレート変動計算。
    * *変える場合*: 係数（24や20）を大きくすると、1試合でのレート変動が激しくなります。
* **`applyScore`**: モルックの基本ルール（50点ぴったりで勝利、50点を超えると25点へ戻る）を定義。

### ③ コンポーネント
#### `GameView` (試合進行エンジン)
試合中の全ての状態（現在のレッグ、投擲順、ミス回数、履歴、**1投ごとのログ**）を管理します。
* **`confirmScore`**: スコア確定時の核となる処理。ここでミス判定、レッグ勝敗、次の投擲者の決定、**`logs`への記録**を行っています。
* **`game.logs`**: 1投ごとの `{who, val, leg, time}` を配列で保持します。

#### `App` (メイン画面・データ同期)
タブの切り替えやFirebaseとの通信を担当します。
* **`useEffect`**: Firebaseの `molkky_data` ドキュメントを監視し、変更があれば即座に画面を更新します。
* **`recordResult`**: 試合終了時に呼び出され、レート更新と全試合データのFirebase送信を行います。

## 3. データ構造 (Firestore)
`tournament/molkky_data` ドキュメント内に以下の形式で保存されます。

```json
{
  "players": { "pId": { "name": "...", "rate": 1500 } },
  "matches": [ { "p1": "...", "p2": "...", "result": "...", "logs": [...] } ],
  "phase": "rate" // "rate" (順位決定) or "league" (本戦)
}
```

## 4. カスタマイズ・学習ガイド

### Q. 順位戦の進出人数を変えたい
`App` コンポーネント内の `tab === 'league'` 部分にある `.slice(0, 8)` の数字を変更してください。ここを変えると、総当たり表に表示される人数が変わります。

### Q. 3連続ミスで失格のルールを変えたい
`GameView` 内の `isDisq = c0[who] >= 3` の数字を変更してください。例えば `4` にすれば4回ミスまで許容されます。

### Q. 1投ごとのログに他の情報（例：狙ったスキトル）を加えたい
1.  `confirmScore` 内の `newLog` オブジェクトに新しいプロパティを追加します。
2.  入力UI（`ScorePanel`付近）に選択肢を追加し、`confirm` 状態に持たせるように改造します。

### Q. レートの計算式をより厳密にしたい
`calcWinGain` 関数内の計算式を、研究室で採用したい統計モデル（例：チェスと同じ本格的なイロレーティング等）に書き換えてください。

---

## 5. 運用上の注意点
* **Firebaseの認証**: 本コードは `compat` 版のSDKを使用しています。将来的にFirebaseのバージョンを上げる場合は、モジュラー形式への書き換えが必要です。
* **スプレッドシート連携**: GAS（Google Apps Script）側は、Firebaseのこのデータ構造を前提に作られています。アプリ側でプロパティ名（`logs`など）を変える場合は、GAS側のコードも同時に修正してください。
